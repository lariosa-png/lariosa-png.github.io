Beacon Engine

Demo Showcase

About

Beacon Engine is a rendering engine developed with C++ OpenGL for the final HND year at ESAT - Valencia, This project was developed by Alberto Lario Santos and Eva Segarra Raro.

Features

  • Engine works on multiple threads, working on rendering while updating the game logic for the following frame. Using Display Lists to perform deferred rendering commands.

  • The engine has been developed following the Entity-Component-System (ECS) philosophy.

  • Beacon engine supports up to four directional lights, all of them casting shadows.

  • Simple scenes can be loaded from a Lua file. However the user will have to use the in game scene creation in order to take advantage of the scene graph.

  • Beacon Engine includes two post processes that the user will be able to activate at will.

Strategies and Solutions to development problems

Find components with systems:

Every component uses an ID to find the components that the system needs we added a function in the system that allows you to add trackers to the system, the system will receive the IDs of the components it needs and add those to a list, after that the ECS manager will use those trackers to give the entities to the system.

Camera and lights similarities for render:

The camera and lights have to use the same variables to be rendered, to be able to call both types of renders with the same command we created a component called viewprojection that calculates the view and the projection of both types of entities, with the render system we can cast into this component and be able to use both types.

Shadow Maps:

In order to generate correct shadow maps, the render system the team had implemented had to be modified to run multiple times on a single frame, making use of the scene lights as cameras. A new system was created to work in conjunction with it and find all light, all of witch would be given to the Render System with their own render buffer to generate the shadow maps. While working on the shadow maps the team faced problems due to a bad management of opengl texture units, the team lost many hours debugging this problem, but once detected was easily solved.
